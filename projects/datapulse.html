DataPulse

Scala Apache Spark Python Flask Docker Compose

Pipeline dati end-to-end che simula un ambiente Big Data reale in locale. Apache Spark (Scala) elabora un dataset CSV eseguendo aggregazioni e trasformazioni, scrive i risultati su file, e Flask li legge per visualizzarli in una dashboard interattiva. L'intera infrastruttura è orchestrata con Docker Compose in tre container separati.

Architettura della pipeline

CSV Input → Spark Job (Scala) → Output JSON/CSV → Flask Dashboard (Python)
orchestrato da Docker Compose · container: spark-processor | flask-dashboard | shared-volume

Funzionalità principali

  • Job Spark in Scala — lettura CSV, pulizia dati, aggregazioni (count, sum, avg) e scrittura risultati
  • Flask dashboard — legge i file di output Spark e li visualizza con tabelle e grafici
  • Docker Compose — tre servizi: spark-processor, flask-dashboard, volume condiviso per i file intermedi
  • Architettura decoupled — il layer di processing (Scala/Spark) è completamente separato dal layer di visualizzazione (Python/Flask)
  • Scala idiomatico — utilizzo di DataFrame API, trasformazioni lazy e ottimizzazioni Catalyst
  • Progetto dimostrativo del pattern reale usato in ambienti Big Data enterprise

Stack tecnico

  • Scala / Apache Spark — job di processing dati con DataFrame API; lettura, trasformazione e scrittura batch
  • Python / Flask — dashboard web leggera che legge i dati elaborati da Spark e li serve come UI
  • Docker Compose — orchestrazione multi-container con volumi condivisi per il passaggio dati tra servizi
  • SBT / Maven — build del progetto Scala con gestione dipendenze Spark
